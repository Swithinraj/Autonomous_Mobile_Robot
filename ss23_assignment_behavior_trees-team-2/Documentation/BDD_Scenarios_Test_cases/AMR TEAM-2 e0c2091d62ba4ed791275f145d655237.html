MS 1 - Assemble Robile and controlling Joypad

US 1:

As a system builder I want the joypad control software to work for diﬀerent hardware  conﬁgurations of the Robile So that the maintenance of the software is simpliﬁed.

Test Case 1:

Given: A Robile is configured and powered on.

When: A joystick signal is sent to the Robile.

Then: The Robile responds by initiating motion, moving accordingly to the joystick commands.

MS 2 - Collision Avoidance

US 1:

As a safety engineer I want the Robile not to run into static or dynamic obstacles So that injuries to humans and damages to the environment and the Robile can be avoided.

Test Case 1 - Static Obstacles

Given: The Robile is in motion,

When: It encounters objects in its path,

Then: Robile avoids obstacles and continues following.

Test Case 2 - Dynamic Obstacles

Given: The Robile is in motion, and there is a possibility of objects entering its path.

When: Objects intersect with the Robile's path during its movement.

Then: Robile should try to avoid collision and stop when too close to the threshold.

MS 3 - Wall Detection and Alignment

US 1 As a behavior developer I want the Robile to be able to find the wall and align to it.

Scenario 1: One wall

Test Case 1:

Given :  Robile and all its systems are running without problems in a static environment.

When:  The Robile is initialized to confront a wall.

Then:  The Robile should align to the wall.

Scenario 2: Glass wall

Test Case 1:

Given :  Robile and all its systems are running without problems in a static environment.

When:  The Robile is initialized to confront a glass wall.

And: The glass wall is the closest wall to the Robile.

Then:  The Robile should align to the glass wall.

Scenario 3: Parallel wall

Objective: Test the Robile's ability to move in a narrow corridor without getting stuck.

Test case 1:

Given :  Robile and all its systems are running without problems.

When:  The Robile is initialized between two parallel walls .

And: The Robile is at equal distance from both.

Then:  The Robile should be able to pick one of them to follow.

Test case 2:

Given :  Robile and all its systems are running without problems.

When:  The Robile is initialized to confront two parallel walls .

And: The Robile is at equal distance from both.

Then:  The Robile should be able to move in the corridor and  to pick one of them to follow.

Scenario 4: No walls

Test Case 1:

Given :  Robile and all its systems are running without problems in a static environment.

When: The Robile is initialized to have no walls within the Robile’s range.

Then: The Robile rotates clockwise or in the direction where the last wall was found.

Scenario 5: Corner

Test Case 1: Closed corner

Given :  Robile and all its systems are running without problems in a static environment.

When:  The Robile is initialized to confront a close corner.

Then:  The Robile should align to the nearest wall.

Test Case 2:

Given :  Robile and all its systems are running without problems in a static environment.

When:  The Robile is initialized to confront a close corner.

And : The two walls are at the same distance.

Then:  The Robile should select one of the walls and align it.

Test Case 3: Open Corner

Given :  Robile and all its systems are running without problems in a static environment.

When:  The Robile is initialized to confront an open corner.

Then:  The Robile should align to the nearest wall.

Test Case 4:

Given :  Robile and all its systems are running without problems in a static environment.

When:  The Robile is initialized to confront an open corner.

And : The two walls are at the same distance.

Then:  The Robile should select one of the walls and align it.

MS 4 - Wall Following

US 4.1 As a behavior developer I want the Robile to be able to follow the detected nearest wall after aligning.

Scenario 1: One Wall

Test Case 1:

Given :  Robile and all its systems are running without problems in a static environment.

When:  The Robile is initialized to confront a wall.

Then:  The Robile should align to the wall and follow it.

Scenario : Glass wall

Test Case 1:

Given :  Robile and all its systems are running without problems in a static environment.

When:  The Robile is following a wall.

And: The glass wall is encountered in a close corner or open corner.

Then:  The Robile should detect the glass wall as a normal wall and behave as in case one and two.

Scenario 3: Slanted Wall

Test Case 1:

Given :  Robile and all its systems are running without problems in a static environment.

When:  The Robile is following a wall.

And: An obstacle is encountered.

Then:  The Robile should detect the obstacle and estimate its geometry and maneuver around it.

Scenario 2: Corner

Test Case 1:

Given :  Robile and all its systems are running without problems in a static environment.

When:  The Robile is following a wall and encounters a close corner.

Then:  The Robile should rotate to follow the new wall.

Test Case 2:

Given :  Robile and all its systems are running without problems in a static environment.

When:  The Robile is following a wall and encounters an open corner.

Then:   The Robile should rotate to follow the new wall.

Scenario 3: Obstacle avoidance

Test Case 1:

Given :  Robile and all its systems are running without problems in a static environment.

When:  The Robile is following a wall

And: An obstacle is encountered.

Then:  The Robile should detect the obstacle and estimate its geometry and maneuver around it
